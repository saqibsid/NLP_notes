Exploratory Data Analysis¶

Exploratory Data Analysis (EDA) in Natural Language Processing (NLP) is the process of visually and statistically analyzing a dataset containing text data to uncover patterns, trends, anomalies, and insights. EDA helps NLP practitioners and data scientists better understand the textual data they are working with before diving into more advanced NLP tasks such as text classification, sentiment analysis, topic modeling, or machine translation.

Extracting Term Frequencies

Term frequency (TF) is a numerical statistic used in natural language processing and text analysis to quantify how frequently a term (word or phrase) appears in a document or a corpus of documents. It's a fundamental concept in text mining and information retrieval. Term frequency is typically calculated using the following formula:

$TF(t,d) = \frac{Number \ of \ times \ term \ t \ appears \ in \ document \ d}{Total \ number \ of \ terms \ in \ document \ d}$¶

WordCloud¶

A word cloud is a visual representation of text data, where the size of each word is proportional to its frequency or importance in a given corpus of text. In a word cloud, more frequent words are typically displayed in a larger font size, making it easy to see which words are most prominent in the text.

Word clouds are often used for data visualization and can help quickly identify the most significant terms or themes in a body of text. They are commonly used in various fields, including natural language processing, text analysis, marketing, and content summarization.

Initializing VADER SIA¶

VADER (Valence Aware Dictionary and sEntiment Reasoner) is a lexicon and rule-based sentiment analysis tool specifically designed for analyzing the sentiment of text in the context of social media and informal online text. It was developed by researchers at the Georgia Institute of Technology.

The output will include scores for positivity, neutrality, and negativity, as well as a compound score that summarizes the overall sentiment of the text.

Application of TF-IDF and N-Gram¶

An n-gram is a contiguous sequence of n items (or words) from a given sample of text or speech. N-grams are used in various natural language processing (NLP) and text analysis tasks to capture and represent the relationship between words or characters within a text. The "n" in n-gram refers to the number of items in each sequence.

Within Sentiment Analysis - N-grams can be used to extract and analyze phrases or idioms that are indicative of sentiment in text data.

What is Model Generalizability ?

Model generalization is a critical concept in machine learning and is closely related to a model's ability to perform well on unseen or new data. When a machine learning model generalizes well, it means that it has learned to capture the underlying patterns in the training data and can apply that knowledge to make accurate predictions or classifications on data it hasn't encountered before. Generalization is a fundamental goal because it indicates that a model has learned useful, transferable knowledge rather than memorizing the training data.

What If Section ?¶

Suppose you wanted to check how different methods such as the ones listed below affect the final scores on our best model

  1. Count Vectorization
  2. TF-IDF (Word-Level Granularity)
  3. TF-IDF (N-Gram Level Granularity)
In [63]:
%%time
def score(method_list):
    scores = []
    for method in method_list:
        rf_model = RandomForestClassifier().fit(method, y)
        scores.append(cross_val_score(rf_model, method, y, cv=5, n_jobs=-1).mean())

    scores_df = pd.DataFrame({"Methods": ["Count Vectorization", "TF-IDF Word-Level", "TF-IDF N-Gram"],
                             "Scores": scores})

    trace1 = go.Bar(
                    y = scores_df["Methods"],
                    x = scores_df["Scores"],
                    name = "Accuracy Plot",
                    text=[round(i,5) for i in scores_df['Scores']],
                    marker = dict(color = ['#2ECC71','#34495E','#D0D3D4'],
                                 line=dict(color='rgb(0,0,0)',width=1.5)),
        orientation='h', textposition = 'inside'
    )
    data = [trace1]
    layout = go.Layout(barmode = "group",
                       title={'text': "Scores" ,
                                 'y':0.9,
                                 'x':0.5,
                                 'xanchor': 'center',
                                 'yanchor': 'top'},
                          template='plotly_dark')
    fig = go.Figure(data = data, layout = layout)
    pio.show(fig)

method_list = [X_count, X_tf_idf_word, X_tf_idf_ngram]
